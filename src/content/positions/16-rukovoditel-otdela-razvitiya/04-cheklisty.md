# РОР: чек-листы

Набор готовых чек-листов, которыми пользуется РОР. Чек-лист = страховка от того, чтобы «забыть» рутину.

---

## Чек-лист 1. Ежедневный (15 пунктов)

Заполняется в конце каждого рабочего дня. Хранится в журнале РОР.

- [ ] Провёл ежедневную планёрку 08:00 с менеджерами развития (все ли были, что был «вопрос дня»)
- [ ] Просмотрел дашборд входной воронки за вчера
- [ ] Прослушал минимум 5 звонков менеджеров развития
- [ ] Написал письменный фидбэк по каждому прослушанному звонку
- [ ] Просмотрел чат с РОП на жалобы по качеству анкет
- [ ] Просмотрел чат с маркетологом по платным каналам
- [ ] Провёл запланированные 1-1 (если были по графику)
- [ ] Проверил выполнение задач, поставленных менеджерам вчера
- [ ] Поставил задачи на завтра каждому менеджеру (через чат / CRM)
- [ ] Заполнил дашборд входной воронки за сегодня (если не сделал менеджер)
- [ ] Подготовился к завтрашней планёрке: знаю «вопрос дня»
- [ ] Записал в свой журнал ключевые наблюдения / инсайты дня
- [ ] Проверил статус активных партнёрских договорённостей (школы, партнёры)
- [ ] Подтвердил вечером выходы промоутеров на завтра (через менеджеров развития)
- [ ] Закрыл день: ответил на все висящие сообщения от команды

**Норма:** ≥ 13/15 пунктов ежедневно. < 11 — сигнал внеочередной ретроспективы.

## Чек-лист 2. Еженедельный (заполняется в пятницу к 18:00)

- [ ] Провёл 5 ежедневных планёрок
- [ ] Участвовал в стратегической планёрке (понедельник 10:00) с подготовленными цифрами
- [ ] Провёл еженедельную обучающую сессию для менеджеров развития (четверг 15:00)
- [ ] Провёл еженедельную разборную планёрку отдела
- [ ] Провёл синк с РОП (1 час)
- [ ] Провёл синк с маркетологом (30–45 мин)
- [ ] Провёл синк с одним из РФ (по графику ротации)
- [ ] Провёл 1-1 с 2 менеджерами развития (норма за неделю)
- [ ] Прослушал ≥ 25 звонков за неделю и оставил фидбэк
- [ ] Свёл недельный дашборд: план/факт по входным анкетам по городам и каналам
- [ ] Подготовил повестку для следующей стратегической планёрки
- [ ] Закрыл все «грязные анкеты» недели в формат разбора (что меняем в скрипте / регламенте)
- [ ] Проверил статус и качество всех активных промо-точек
- [ ] Подготовил план следующей недели для каждого менеджера развития

**Норма:** ≥ 12/14 пунктов еженедельно.

## Чек-лист 3. Ежемесячный

Заполняется в первые 3 рабочих дня нового месяца.

- [ ] Свёл ежемесячный дашборд: план/факт по всем городам и каналам
- [ ] Свёл стоимость лида (CAC) по каждому каналу
- [ ] Подготовил аналитический отчёт собственнику / ОД (выводы и решения)
- [ ] Презентовал результаты отдела собственнику / ОД
- [ ] Закрыл премии / штрафы за прошлый месяц (для менеджеров развития и промоутеров)
- [ ] Проверил по всем 7 городам — есть ли неотработанные/«протухшие» анкеты, что с ними
- [ ] Аудит партнёрств: какие школы и партнёры активны, какие «остыли»
- [ ] Запланировал минимум 1 выезд в город (если не Челны) на месяц
- [ ] Если месяц завершает квартал — провёл аттестацию (по плану раз в полугодие)
- [ ] Обновил квартальный план (если квартал заканчивается / начинается)
- [ ] Проверил по всем каналам — кто в зоне риска по объёму / качеству, что делаем
- [ ] Обновил карту школ и партнёров (актуализация статусов)

**Норма:** ≥ 10/12 пунктов ежемесячно.

## Чек-лист 4. Качество звонка менеджера развития (для прослушивания)

10-балльная шкала. Заполняется по каждому прослушанному звонку.

| # | Параметр | Балл (0–10) |
|---|---|---|
| 1 | Открытие звонка: представился, обозначил повод | |
| 2 | Подтвердил основные данные: возраст ребёнка, кто звонит |  |
| 3 | Заполнил поле «школа, в которой обучается» | |
| 4 | Заполнил поле «адрес проживания» | |
| 5 | Зафиксировал «ближайшая локация» | |
| 6 | Корректно квалифицировал интерес (горячий / тёплый / холодный) | |
| 7 | Согласовал удобное время для звонка хантера | |
| 8 | Отработал возражение (если было) корректно | |
| 9 | Тон, эмпатия, темп — соответствуют скрипту | |
| 10 | Завершил с чётким следующим шагом | |
| | **Итоговый средний балл** | |

**Зоны:**
- 🟢 8.0–10.0 — отличный звонок, фиксируем как «эталон»
- 🟡 6.0–7.9 — есть точки роста, фидбэк менеджеру + добавить в ролевые игры
- 🔴 < 6.0 — критический разбор лично с менеджером в течение 24 часов

## Чек-лист 5. Аудит промо-точки (выезд / по фото)

Используется при выезде в город или при ежемесячном фото-отчёте от менеджера развития.

### Локация и условия

- [ ] Точка находится в правильном месте (поток, видимость, проход родителей с детьми)
- [ ] Время работы точки совпадает с пиковым потоком целевой аудитории
- [ ] Точка визуально оформлена (брендирование, ростовая фигура / стенд / баннер)
- [ ] Соблюдены договорённости с площадкой (ТРЦ, парк, школа)

### Промоутер

- [ ] Промоутер в форме / с брендированной атрибутикой
- [ ] Промоутер выглядит опрятно, не сидит в телефоне
- [ ] Промоутер активно работает (подходит к родителям, не ждёт пассивно)
- [ ] Промоутер презентует продукт по скрипту (заслушивание выборочно)
- [ ] Промоутер заполняет анкету корректно (все обязательные поля)
- [ ] Промоутер передаёт анкеты в CRM в течение часа после получения

### Метрики точки за неделю

- [ ] Количество собранных анкет соответствует норме (для этой точки)
- [ ] Конверсия анкета → запись на пробное ≥ 35%
- [ ] Стоимость лида (затраты на промоутера + площадку / количество анкет) в норме

### Итог

| Зона | Решение |
|---|---|
| 🟢 Всё в порядке | Точку оставляем, поощряем промоутера |
| 🟡 Есть слабые места | Корректировка с менеджером развития, повторная проверка через 2 недели |
| 🔴 Системные проблемы | Заменить промоутера / перенести точку / закрыть точку |

## Чек-лист 6. Запуск нового канала лидгена

Используется при тестировании любого нового канала (новая школа, новая площадка, новый формат мероприятия, новый партнёр).

### Подготовка (до запуска)

- [ ] Описана гипотеза: какой сегмент ЦА, ожидаемый объём анкет в месяц, ожидаемая стоимость лида
- [ ] Определён бюджет (если есть) и контрольная точка остановки (если результат < X — закрываем)
- [ ] Назначен ответственный (менеджер развития города или сам РОР для пилота)
- [ ] Подготовлены материалы (скрипт, форма анкеты, презентация / промо)
- [ ] Согласована площадка / партнёр (договорённость зафиксирована письменно)

### Запуск (первые 2 недели)

- [ ] Старт прошёл по плану — все материалы / люди на месте
- [ ] Ведётся ежедневный учёт анкет с этого канала отдельно (тег / источник в CRM)
- [ ] Качество анкет проверяется (минимум 5 анкет вручную)
- [ ] Передача в РОП организована, РОП в курсе нового канала

### Точка контроля (30 дней)

- [ ] Фактические показатели сравнены с гипотезой
- [ ] Принято решение: **масштабировать / докрутить / закрыть**
- [ ] Решение зафиксировано в журнале каналов, обсуждено с собственником

## Чек-лист 7. Передача анкеты от менеджера развития в РОП

Используется как «золотой стандарт», обучающий материал для менеджеров развития.

Перед передачей анкеты хантеру:

- [ ] В карточке клиента заполнены все обязательные поля: ФИО ребёнка, ФИО родителя, телефон, возраст ребёнка
- [ ] Заполнено поле «школа, в которой обучается»
- [ ] Заполнено поле «адрес проживания» (минимум — район / ЖК)
- [ ] Заполнено поле «ближайшая локация» KIBERone
- [ ] Зафиксирован канал получения анкеты (промоутер / школа / сайт / реклама / мероприятие)
- [ ] Зафиксирован сегмент: горячий / тёплый / холодный
- [ ] Согласовано удобное время для звонка хантера (если было обсуждено)
- [ ] Записан комментарий менеджера развития: чем интересовался родитель, какие сомнения

**Анкета без любого из обязательных пунктов — НЕ передаётся в РОП.** Менеджер развития доквалифицирует и передаёт после.

**Если менеджер развития передал анкету без обязательных полей → автоматический разбор с РОР, повторный случай → депремирование.**
