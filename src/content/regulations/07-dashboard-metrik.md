# Дашборд ключевых метрик KIBERone-сети

**Цель:** еженедельный/ежемесячный взгляд собственника на здоровье каждого филиала. Без этого все «процессы» работают вслепую.

**Где живёт:** Google Sheets (рекомендую, чтобы команда видела). Альтернатива — Excel в OneDrive.

**Кто заполняет:**
- Фермер своего города — в первый понедельник каждого месяца
- Помощник-директор сводит общую картину — до среды
- Собственник просматривает на еженедельной планёрке

---

## СТРУКТУРА ТАБЛИЦЫ

### Лист 1 — «Месячные метрики по городам»

Колонки:

| Колонка | Что записываем | Источник |
|---|---|---|
| Месяц | Январь 2026, Февраль 2026… | Вручную |
| Город | Челны / Казань / … | Вручную |
| База на 1-е число | Сколько активных учеников | CRM s20.online |
| Новых пришло | Сколько новых клиентов в этом месяце | CRM |
| Ушло | Сколько перестало платить | CRM |
| База на 30-е число | = база на 1-е + новые - ушли | Формула |
| Месячный отток, % | = ушло / база на 1-е × 100 | Формула |
| Месячный приток, % | = новые / база на 1-е × 100 | Формула |
| Чистый рост, % | = приток - отток | Формула |
| Пробных уроков проведено | За месяц | CRM |
| Конверсия пробного, % | = новых / пробных × 100 | Формула |
| Стоимость лида, ₽ | Маркетинговый бюджет / лидов | Маркетинг |
| Стоимость клиента, ₽ | = марк.бюджет / новых | Формула |
| Выручка месяца, ₽ | Сумма платежей | CRM |
| % продления (только август) | Из закончивших какие записались | CRM |

Заполняется фермерами **еженедельно** в понедельник. Это рабочий список — кому нужно срочное внимание.

### Лист 4 — «Когортный анализ» (на потом, когда появятся 6+ месяцев данных)

Когорта = группа учеников, начавших обучение в одном месяце. Смотрим, сколько остались через 1, 3, 6, 12 месяцев. Это покажет:
- Реальный LTV
- Где «обрывается» когорта
- Эффект изменений (после внедрения отчёта прогресса когорты должны держаться лучше)

## КЛЮЧЕВЫЕ КОММЕНТАРИИ ПО ВНЕДРЕНИЮ

### 1. Не делайте «дашборд ради дашборда»
- Если данных в CRM ещё нет в нужном виде — сначала наладьте сбор, потом дашборд
- Лучше 5 ключевых метрик регулярно, чем 30 раз в полгода

### 2. Сначала минимум, потом расширяйте
**Минимум для месяца 1-2:**
- База учеников на 1-е и 30-е
- Новые / ушли
- Выручка

Этого достаточно, чтобы видеть отток/приток. Конверсии и когорты — следующий этап.

### 3. Кто за что отвечает
- **Фермер города** — заполняет данные по своему городу до 3-го числа следующего месяца
- **Помощник-директор** — собирает агрегаты до 5-го числа, выводит в дашборд собственника
- **Собственник** — смотрит на еженедельной планёрке, реагирует на красные зоны

### 4. Реагирование на красные сигналы

| Сигнал | Реакция |
|---|---|
| Город 1 месяц в красной зоне | Запрос пояснений у фермера + тьюторов |
| Город 2 месяца подряд | Антикризисная встреча с командой города |
| Город 3 месяца подряд | План действий с deadline 30 дней |

### 5. Что делать ОБЯЗАТЕЛЬНО
- Дашборд **должен попадать на ежемесячный отчёт собственнику**
- Цифры **никогда не корректируются «потому что сезонность»** — фиксируем как есть
- При запуске нововведений **отмечать дату внедрения** прямо в дашборде, чтобы видеть эффект

## ОЖИДАЕМЫЙ ЭФФЕКТ ОТ ВНЕДРЕНИЯ ДАШБОРДА

- Раннее обнаружение проблем (не «упало 3 раза → паника», а «пошло вниз → реакция»)
- Объективные основания для управленческих решений (по Краснодару — увидите «сценарий А или Б» через 2-3 месяца не догадываясь, а зная)
- Прозрачность для команды → их ответственность за результат
- Возможность ставить **числовые цели** менеджерам и руководителям отделов

**Стоимость внедрения:** ~5 часов работы (помощник-директор + вы для согласования). 0 рублей.

**Эффект:** косвенный, но сквозной — все ваши решения станут на порядок точнее.
