# Чек-листы РОП

Все чек-листы — рабочий инструмент. Заполнение фиксируется (галочка с датой), хранится минимум 12 месяцев.

**Логика:** РОП управляет 3-7 хантерами и работает со скриптами, которые меняются. Без чек-листов он либо тонет в оперативке, либо пропускает шлифовку скриптов / прослушивания.

---

## ЧЕК-ЛИСТ 1. Ежедневный (заполняет в конце дня)

**Дата:** _______ **РОП:** _______

### Управление командой
- [ ] Утренняя планёрка с хантерами проведена (15 минут)
- [ ] Чат с хантерами проверен 2-3 раза за день
- [ ] Если были эскалации — обработаны или в работе

### Прослушивание и обучение
- [ ] Прослушано минимум 10-15 звонков за день (от разных хантеров)
- [ ] Каждому хантеру отправлен фидбэк по минимум 1-2 звонкам
- [ ] Если есть критическая ошибка — личный звонок хантеру

### Аналитика
- [ ] Дашборд продаж проверен утром (план/факт)
- [ ] Если конверсия упала за вчера — есть гипотеза почему

### Координация
- [ ] Чат с менеджерами развития / фермерами / маркетологом проверен
- [ ] Если поступил вопрос — отвечен в течение 4 часов

### Документы
- [ ] Заметки за день внесены в дневник РОП
- [ ] План на завтра составлен

**Если хотя бы одна галочка не стоит** — день не закрыт. На следующий день — приоритет догнать.

## ЧЕК-ЛИСТ 2. Недельный (заполняет в пятницу)

**Неделя с _____ по _____** **РОП:** _______

### Прослушивание и фидбэк
- [ ] Каждый хантер прослушан минимум 10 раз за неделю
- [ ] Каждый получил письменный фидбэк
- [ ] 1-1 встречи: минимум с 2-3 хантерами за неделю

### Скрипты
- [ ] Если были обновления — обучены хантеры
- [ ] Если есть гипотеза по новой формулировке — протестирована в ролёвках

### Аналитика
- [ ] Конверсия по каждому хантеру за неделю — посчитана
- [ ] Распределение хантеров по зонам (зелёная / жёлтая / красная) — обновлено
- [ ] Топ-3 типа отказов за неделю — выделены

### Координация
- [ ] Синк с менеджерами развития / фермерами — был или запланирован
- [ ] Если есть жалобы на качество анкет / клиентов — переданы

### Большая пятничная планёрка
- [ ] Подготовлена и проведена
- [ ] Топ-успехи и топ-провалы озвучены
- [ ] План на следующую неделю объявлен

### Документы
- [ ] Дашборд недели заполнен
- [ ] Отчёт за неделю отправлен собственнику / ОД

## ЧЕК-ЛИСТ 3. Ежемесячный (последняя пятница)

**Месяц _____** **РОП:** _______

### Метрики (см. `05-pokazateli-i-istochniki.md`)
- [ ] Общая выручка нового бизнеса за месяц: ___ ₽ (план ___)
- [ ] Количество новых продаж: ___
- [ ] Конверсия пробное → продажа: ___% (норма ≥ 45%)
- [ ] Средний чек: ___ ₽
- [ ] Скорость обработки анкеты (среднее): ___ часов
- [ ] Распределение хантеров: 🟢 ___, 🟡 ___, 🔴 ___

### Аттестация хантеров
- [ ] Если в этом месяце была аттестация — проведена
- [ ] Решения по красной зоне приняты (план развития или замена)

### Скрипты
- [ ] Если были обновления — задокументированы
- [ ] Версия скриптов в общей папке актуализирована

### Координация
- [ ] Минимум 1 разбор «грязных» анкет с менеджерами развития

### Презентация собственнику / ОД
- [ ] Презентация месяца подготовлена
- [ ] Встреча проведена
- [ ] План на следующий месяц согласован

### Самооценка
- [ ] Что улучшил, что не получилось, что планирую

## ЧЕК-ЛИСТ 4. Качество звонка хантера (главный чек-лист)

**Дата:** _____ **Хантер:** _____ **Тип звонка:** приглашение / в день пробного / дожим / возражения **РОП:** _____

### Структура звонка (3 пункта)
1. [ ] Хантер представился по стандарту, имя клиента названо
2. [ ] Удержание внимания в первые 30 секунд (нет «холодного» открытия)
3. [ ] Логичная структура: вступление → выявление потребности → презентация → закрытие

### Работа с клиентом (4 пункта)
4. [ ] Хантер слушал больше, чем говорил (норма: клиент говорит 60%+ времени)
5. [ ] Задавал открытые вопросы (не только «да/нет»)
6. [ ] Реагировал на сомнения клиента (не игнорировал)
7. [ ] Использовал имя ребёнка несколько раз

### Скрипт (3 пункта)
8. [ ] Соблюдал актуальный скрипт (не «отсебятина»)
9. [ ] Если возражение — отработал по скрипту (или достойно если выходил за скрипт)
10. [ ] Закрытие: конкретное предложение со сроком (не «подумайте»)

### Итого баллов: ___ / 10

**Шкала:**
- 9-10 — отличный звонок, эталон для команды
- 7-8 — нормальный звонок, есть зоны роста
- 5-6 — есть проблемы, нужен план развития
- ≤ 4 — критический звонок, эскалация + срочное обучение

_______________________________________________

**Фидбэк хантеру отправлен:** [ ] Да, в _______ [ ] Нет

## ЧЕК-ЛИСТ 5. 1-1 с хантером (раз в 2 недели)

**Дата:** _____ **Хантер:** _____ **Длительность:** 30-45 минут

### Подготовка (за 10 минут)
- [ ] Прочитал статус хантера (KPI за 2 недели)
- [ ] Просмотрел оценки прослушанных звонков
- [ ] Подготовил 2-3 темы для обсуждения

### Структура встречи

**1. Как дела (5-10 минут)**
- [ ] Спросил, как чувствует себя в работе
- [ ] Слушал, не «руководил»

**2. Текущие сложные кейсы (10-15 минут)**
- [ ] Какие клиенты «висят»
- [ ] Что не получается
- [ ] Совместный разбор стратегии

**3. Обратная связь по звонкам (10-15 минут)**
- [ ] Озвучил конкретные наблюдения с прослушанных звонков
- [ ] Похвалил за конкретные удачные моменты
- [ ] Указал на конкретные зоны роста (1-2, не больше)

**4. Развитие (5-10 минут)**
- [ ] Что хочет улучшить
- [ ] Что нужно от меня

**5. Договорённости (5 минут)**
- [ ] Конкретные действия с обеих сторон, с датами

### После встречи
- [ ] Записал ключевые моменты в дневник
- [ ] Если есть запросы — поставил себе задачу

## ЧЕК-ЛИСТ 6. Аттестация хантера (раз в полугодие)

**Хантер:** _____ **Дата:** _____ **Стаж:** ___ месяцев

### Метрики за 6 месяцев
- [ ] Конверсия пробное → продажа: ___% (норма ≥ 45%)
- [ ] Средний чек: ___ ₽
- [ ] Количество продаж: ___
- [ ] Средний балл звонков по чек-листу 4: ___ /10 (норма ≥ 7.5)
- [ ] Жалоб от клиентов / фермеров: ___ (норма ≤ 2)

### Качественная оценка
- [ ] Соблюдает скрипты
- [ ] Растёт по фидбэкам (не повторяет одни и те же ошибки)
- [ ] Хороший командный игрок

### Самооценка хантера
- [ ] Хантер заполнил мини-опрос: что получается, что хочет улучшить

### Решение
- [ ] **Зелёная зона:** продолжаем, рассмотреть рост (старший хантер / повышение)
- [ ] **Жёлтая зона:** план развития на 3 месяца + контрольная аттестация
- [ ] **Красная зона:** план «исправить или замена» — 1 месяц

**Подписи:** РОП _____, собственник (для красной зоны) _____

## ЧЕК-ЛИСТ 7. Обновление скрипта

**Скрипт:** _____ **Дата обновления:** _____ **Причина:** _____

### Подготовка
- [ ] Проблема ясно сформулирована (например: «60% клиентов отказываются на возражении 'дорого'»)
- [ ] Прослушано минимум 20 звонков с этим возражением
- [ ] Гипотеза новой формулировки готова

### Тестирование
- [ ] Ролёвки с командой (минимум 5 ролёвок)
- [ ] Корректировка по результатам
- [ ] Пилот: 2-3 хантера тестируют в реальных звонках 1 неделю
- [ ] Замер результата (конверсия с этим возражением)

### Внедрение
- [ ] Если пилот успешен — обучение всей команды
- [ ] Старая версия скрипта в архив, новая — в общую папку
- [ ] Обновлены все связанные материалы (дашборд, чек-листы)

### Контроль
- [ ] Через 2 недели — замер: помогает ли новый скрипт
- [ ] Если не помогает — откат к старой версии или новая итерация

**Кандидат:** _____ **Дата:** _____

### Подготовка
- [ ] Согласована вакансия с собственником / ОД
- [ ] HR подключён к процессу

### Собеседование
- [ ] Прочитал резюме
- [ ] Подготовил вопросы под роль (минимум 7-10)
- [ ] Длительность не менее 60 минут
- [ ] Тестовое задание: ролёвка-разговор (РОП в роли клиента)

### Решение
- [ ] Согласовано с собственником / ОД
- [ ] Оффер сделан в срок

### После найма
- [ ] Запущена программа онбординга (3-4 недели)
- [ ] Назначен наставник из числа сильных хантеров
- [ ] Все доступы / документы готовы

## ЧЕК-ЛИСТ 9. Месячный синк с собственником / ОД

**Дата:** _____ **Месяц:** _____

### Подготовка (за 24 часа до)
- [ ] Презентация готова
- [ ] Все KPI зафиксированы
- [ ] Список «нужно решений» — составлен

### Структура встречи

**1. Результаты месяца (15 мин)**
- [ ] Выручка, продажи, конверсия, средний чек
- [ ] По каждому хантеру (рейтинг)

**2. Главное за месяц (10 мин)**
- [ ] Что хорошо
- [ ] Что плохо
- [ ] Что неожиданно

**3. Скрипты (10 мин)**
- [ ] Что обновили, что планируем
- [ ] Гипотезы по конверсии

**4. Команда (10 мин)**
- [ ] Кто в зелёной / жёлтой / красной зоне
- [ ] Найм / увольнения

**5. Открытые вопросы (15 мин)**
- [ ] Что нужно решить совместно

### После встречи
- [ ] Протокол отправлен в течение 24 часов
- [ ] Все договорённости — в работу

## ЧЕК-ЛИСТ 10. Прослушивание звонков (массовое, раз в неделю)

**Неделя:** _____

### Цель
Прослушать минимум 10 звонков от **каждого** хантера за неделю.

### Распределение
- [ ] Хантер 1: ___ звонков, средний балл: ___
- [ ] Хантер 2: ___ звонков, средний балл: ___
- [ ] Хантер 3: ___ звонков, средний балл: ___
- [ ] (и т.д. по всем)

### Что записываем
- [ ] Топ-3 успешных звонка недели → как кейс для команды
- [ ] Топ-3 проваленных звонка недели → как разбор (без имён)
- [ ] Повторяющиеся ошибки → план обучения

### Обмен с КК
- [ ] Если есть отдельный КК — синк раз в неделю по выявленным ошибкам
- [ ] Согласование «единой картины» (РОП + КК видят одни и те же проблемы)

## КАК ХРАНЯТСЯ ЧЕК-ЛИСТЫ

**Где:**
- Google Drive: `KIBERone Management / Чек-листы РОП / [год] / [месяц] /`
- Каждый чек-лист — отдельный файл с датой
- Раз в квартал — архивируется

**Доступ:**
- РОП — пишет
- Собственник / ОД — читают
- HR — читает аттестации
- Хантеры — видят свои собственные оценки звонков

**Срок хранения:**
- Ежедневные / недельные — 12 месяцев
- Месячные — 24 месяца
- Аттестации — бессрочно
- Оценки звонков — 12 месяцев

**Правило:** если чек-лист не заполнен или потерян, считается, что соответствующая работа НЕ была сделана.
