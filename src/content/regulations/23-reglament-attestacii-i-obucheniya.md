# Регламент аттестации и обучения сотрудников

**Версия:** 1.0
**Дата введения:** 2026-05-22
**Действует для:** всех руководителей отделов KIBERone-сети

---

## Принцип

В KIBERone-сети действует **двухконтурная система развития команд**:

1. **Аттестация** — регулярная оценка квалификации каждого сотрудника **не реже чем раз в полгода**. Результат — объективная картина, кто в зелёной/жёлтой/красной зоне.
2. **Обучение** — постоянный процесс, **раз в неделю или раз в две недели**, по всем сотрудникам своего отдела. Не «по факту проблем», а **планово и непрерывно**.

**За оба контура отвечает руководитель отдела.** Без этого команда не растёт, а руководитель не управляет — он только тушит пожары.

---

## Контур 1. Аттестация (раз в полугодие)

### Принцип

**Каждый сотрудник каждого отдела проходит формальную аттестацию минимум раз в 6 месяцев.** Это не «опросник для галочки», а оценка по понятной шкале с реальными последствиями.

### Кто проходит и кто проводит

| Сотрудник | Кто проводит аттестацию |
|---|---|
| Хантер | **РОП** (руководитель отдела продаж) |
| Фермер | **РОС** (руководитель отдела сопровождения) |
| Менеджер развития (в каждом городе) | **РОР** (руководитель отдела развития) — основной; РФ участвует по городскому KPI |
| Промоутер | РОР через менеджера развития города |
| Тьютор | **Руководитель отдела преподавания** (центральный методист) |
| Ассистент | Руководитель отдела преподавания + РФ (по дисциплине в филиале) |
| Контроль качества звонков | РОС / РОП (тот, кто непосредственно работает) |
| Офис-менеджер | РФ |
| HR | Собственник / ОД |
| Маркетолог | Собственник / ОД |
| РФ | Собственник / ОД |
| РОП / РОС / РОР / руководитель отдела преподавания | Собственник / коммерческий директор |

**Ключевой принцип:** руководители всех четырёх профильных отделов — **РОП, РОС, РОР (отдел развития) и руководитель отдела преподавания** — несут одинаковую полугодовую обязанность по аттестации своих сотрудников и одинаковую обязанность по постоянному (раз в неделю) обучению. Их собственная аттестация — на уровне собственника / коммерческого директора.

### Периодичность

- **Базово:** раз в 6 месяцев
- **Дополнительная** (внеочередная) аттестация — по решению руководителя в случаях:
  - Резкое падение показателей сотрудника
  - Жалобы клиентов / коллег
  - Конфликтная ситуация
  - Перед повышением / переводом

### Что оценивается (структура аттестации)

Каждый отдел разрабатывает свой **аттестационный лист** с 3 блоками:

1. **Hard skills (знание процессов, скриптов, инструментов)** — тест/устный опрос
2. **Practical skills (работа в роли)** — ролевые игры, прослушивание звонков, открытый урок и т.д.
3. **Results (фактические показатели за период)** — KPI сотрудника за последние 6 месяцев

Каждый блок оценивается по 10-балльной шкале. Финальный балл — средневзвешенный.

### Зоны по результату аттестации

| Зона | Балл | Что делает руководитель |
|---|---|---|
| **🟢 Зелёная** | 8.0–10.0 | Подтверждение в должности, обсуждение роста (повышение, расширение зоны, наставничество новичкам) |
| **🟡 Жёлтая** | 6.0–7.9 | Индивидуальный план развития на 3 месяца + усиленное обучение + повторная аттестация через 3 мес |
| **🔴 Красная** | < 6.0 | План на 30–60 дней с конкретными точками контроля. При неулучшении — увольнение или перевод |

### Артефакты аттестации

После каждой аттестации руководитель оформляет:

- **Аттестационный лист** (балл по каждому блоку, итог, зона)
- **Письменный фидбэк сотруднику** (сильные стороны, точки роста, конкретные действия)
- **План развития** (для жёлтой и красной зон)
- **Запись в журнал аттестаций отдела** — общий документ, доступный собственнику / ОД

### KPI руководителя по аттестации

- 100% сотрудников отдела имеют аттестацию не старше 6 месяцев
- Доля «зелёных» сотрудников ≥ 70% от штата
- Все сотрудники из жёлтой/красной зоны имеют **действующий план развития** с точками контроля

## Контур 2. Постоянное обучение (раз в неделю / раз в две недели)

### Принцип

**Обучение в KIBERone — это не разовое мероприятие, а постоянный процесс.** Каждый руководитель отдела проводит со своей командой регулярные обучающие сессии — **раз в неделю или раз в две недели** (в зависимости от размера команды и насыщенности повестки).

Без регулярного обучения:
- Скрипты устаревают, новые продукты не доходят до сотрудников
- Слабые сотрудники не подтягиваются — копится разрыв со «звёздами»
- Команда не понимает изменений в продукте, рынке, конкурентах

### Частота и формат

| Размер команды | Рекомендуемая частота |
|---|---|
| ≤ 3 человека | Раз в 2 недели, 60 минут |
| 4–7 человек | **Раз в неделю**, 45–60 минут (стандарт по умолчанию) |
| 8+ человек | Раз в неделю + дополнительные точечные сессии для подгрупп |

**Минимум для всех отделов сети — раз в 2 недели.** Реже — недопустимо.

### Типовая программа обучающей сессии

| Блок | Что в нём |
|---|---|
| **Обновления (5–10 мин)** | Что нового за период: новый скрипт, новый продукт, новый регламент, изменения в CRM |
| **Разбор кейса (15–20 мин)** | Один конкретный кейс из практики недели: успешный или провальный. Прослушивание звонка / разбор урока / разбор переписки |
| **Ролевая игра / отработка (15–20 мин)** | Парная или групповая отработка нового скрипта / возражения / ситуации |
| **Q&A и закрытие (5–10 мин)** | Вопросы команды, фиксация следующего шага |

### Особенности обучения по отделам

- **Хантеры:** прослушивание звонков + ролевые игры + новые возражения. Ведёт **РОП**
- **Фермеры:** разбор «удержательных» кейсов + работа с пропусками + продление. Ведёт **РОС**
- **Менеджеры развития (отдел развития):** разбор воронки промоутеров + работа с входящими + квалификация + школы и партнёры города. Ведёт **руководитель отдела развития (РОР)**. РФ привлекается для городского контекста, но методическое ведение — на РОР
- **Промоутеры:** скрипты на улице, отработка отказов на промо-точке, работа с QR/анкетой. Ведёт **РОР** через менеджера развития города
- **Тьюторы (отдел преподавания):** методические разборы уроков + новые модули программы + работа с трудными детьми + WOW-эффект на пробном. Ведёт **руководитель отдела преподавания (центральный методист)**
- **Ассистенты:** работа с расписанием + скрипты «у стойки» + взаимодействие с родителями. Ведёт руководитель отдела преподавания (методическая часть) + РФ (дисциплина в филиале)

### Обязательное усиление обучения для жёлтой/красной зоны

Сотрудник, попавший в жёлтую или красную зону по итогам аттестации, получает **дополнительные обучающие сессии**:

- Жёлтая зона: +1 индивидуальная сессия в неделю с руководителем (1-1, разбор персональных слабостей) сверх группового обучения
- Красная зона: +2 индивидуальные сессии в неделю + назначенный наставник из «зелёных» сотрудников. Точка контроля — каждые 2 недели

**Это критично:** без целевого обучения слабый сотрудник не вытянет — групповая сессия раз в неделю его не «починит».

### KPI руководителя по обучению

- Доля проведённых обучающих сессий ≥ 90% запланированных в квартал
- 100% обновлений (новые скрипты, продукты, регламенты) доводятся до команды в течение 7 дней после релиза
- 100% сотрудников жёлтой/красной зоны получают дополнительные индивидуальные сессии

## Журнал обучения и аттестаций отдела

Каждый отдел ведёт **общий журнал** (Google Sheets / Notion) со следующими вкладками:

1. **Аттестации:** дата, сотрудник, балл по блокам, зона, статус плана развития
2. **Обучающие сессии:** дата, тема, кто присутствовал, материалы (ссылка), результат
3. **Индивидуальные обучения:** сотрудник, наставник (если есть), периодичность, точки контроля

Журнал доступен собственнику / ОД для контроля.
