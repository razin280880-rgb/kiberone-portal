# Показатели операционного директора: что измеряем и где брать данные

«Таблица истины» для ОД. Все показатели, по которым оценивается работа ОД.

**Принцип:** ОД — это **управленец всей сети**, поэтому его показатели — это **сводные показатели всей сети** + специфические показатели «как ОД разгружает собственника».

**Главный смысл показателей ОД:** не просто «как идут дела в компании», а «**стало ли лучше под управлением ОД, чем было без него**».

---

## КЛАСТЕР 1. Финансы всей сети (главный кластер)

### 1.1 Выручка всей сети

**Что:** общая выручка по 7 филиалам + штаб.

**Норма:** ≥ 95% от плана.

**Источник:** бухгалтерия / 1С + CRM s20.online.

**Периодичность:** ежемесячно.

**Декомпозиция:**
- По филиалам (7 строк)
- По типам выручки (абонементы / допродажи / прочее)

### 1.2 Прибыль всей сети

**Что:** выручка − все расходы (включая ФОТ, аренду, маркетинг, прочее).

**Норма:** ≥ 90% от плана. Маржа ≥ 25%.

**Источник:** P&L.

**Периодичность:** ежемесячно.

**Тренд:** рост маржи на 1-2 п.п. в год — главный показатель эффективности ОД.

### 1.3 ФОТ как доля от выручки (по всей сети)

**Что:** общий ФОТ (зарплаты + премии + налоги) / выручка.

**Норма:** ≤ 50%.

**Источник:** бухгалтерия.

**Периодичность:** ежемесячно.

### 1.4 Соотношение филиалов в зелёной/жёлтой/красной зоне

**Что:** распределение филиалов по финансовому здоровью.

**Норма:** ≥ 5 из 7 филиалов в зелёной зоне (≥ план), ≤ 1 в красной (≤ 80% плана).

**Источник:** P&L по филиалам.

**Периодичность:** ежемесячно.

**Тренд:** перевод филиалов из красной в зелёную зону — главный показатель работы ОД.

---

## КЛАСТЕР 2. Операционные KPI всей сети (агрегаты по филиалам)

Эти показатели — **сводные** из дашбордов соответствующих ролей. ОД отвечает за **общий уровень**, а не за конкретный город.

### 2.1 База учеников (всего по сети)

**Норма:** рост на ≥ 10% в год для зрелых филиалов, ≥ 30% в год для растущих.

**Источник:** CRM (агрегат).

**Периодичность:** ежемесячно.

### 2.2 Месячный отток (агрегат)

**Норма:** ≤ 8% (целевой ≤ 5%).

**Источник:** CRM (агрегат отписавшихся).

**Периодичность:** ежемесячно.

### 2.3 Доля голосовых звонков фермеров (главный текущий KPI отдела сопровождения)

**Норма:** ≥ 70% по сети.

**Источник:** дашборд фермеров (агрегат).

**Периодичность:** еженедельно.

### 2.4 Лидген (анкеты в месяц по сети)

**Норма:** ≥ 1 200 анкет в месяц по 7 городам в сезон.

**Источник:** дашборды менеджеров развития (агрегат).

**Периодичность:** ежемесячно.

### 2.5 Конверсия пробное → продажа (по сети)

**Норма:** ≥ 45%.

**Источник:** CRM (агрегат).

**Периодичность:** ежемесячно.

### 2.6 Качество (балл методиста по сети)

**Норма:** ≥ 7.5 средний по сети, ≥ 7.0 в любом отдельном филиале.

**Источник:** дашборд методиста.

**Периодичность:** ежемесячно.

### 2.7 Доля продлений на новый учебный год (по сети)

**Норма:** ≥ 75%.

**Источник:** CRM.

**Периодичность:** ежегодно (по итогам апреля-мая).

## КЛАСТЕР 3. Управление командой (специфические для ОД)

### 3.1 Все 7 городов имеют действующего РФ

**Что:** во всех городах есть РФ (не «и.о.»).

**Норма:** 7/7.

**Источник:** оргструктура.

**Периодичность:** ежемесячно.

### 3.2 Текучка управленческого слоя

**Что:** доля ключевых руководителей (РФ + центральные функции = 12 человек), ушедших за период.

**Норма:** ≤ 1 человек в год.

**Источник:** HR.

**Периодичность:** ежегодно.

### 3.3 НПС команды (раз в полугодие)

**Что:** «По шкале 1-10, насколько ты готов рекомендовать KIBERone как место работы?»

**Норма:** средний балл ≥ 7.5 по всей сети.

**Источник:** анонимный опрос через HR.

**Периодичность:** раз в 6 месяцев.

### 3.4 Текучка тьюторов / ассистентов / промоутеров (общая)

**Норма:** ≤ 20% в год.

**Источник:** HR.

**Периодичность:** ежеквартально.

### 3.5 Время заполнения ключевой вакансии

**Норма:** ≤ 8 недель для ключевых ролей.

**Источник:** HR.

**Периодичность:** при каждом найме.

---

## КЛАСТЕР 4. Разгрузка собственника (главный «специфический» KPI ОД)

Это **главное, чем оценивается ОД**. Без этого роль теряет смысл.

### 4.1 Доля времени собственника на оперативке

**Что:** какую долю своего времени собственник тратит на оперативные задачи (планёрки фермеров, разруливание мелких конфликтов, решение повседневных вопросов).

**Целевое:**
- При найме ОД: ~70-80% (текущая ситуация)
- Через 3 месяца: ≤ 50%
- Через 6 месяцев: ≤ 30%
- Через 12 месяцев: ≤ 20%

**Источник:** субъективная оценка собственника (раз в месяц).

**Периодичность:** ежемесячно (через анкету).

**Это самый важный показатель работы ОД.** Если эта цифра не падает — ОД не справляется со своей главной задачей.

### 4.2 Доля решений, которые принимает ОД (без собственника)

**Что:** из 100 операционных решений (расходы, наймы, планы, корректировки) — какую долю принимает ОД самостоятельно.

**Целевое:**
- Через 3 месяца: ≥ 50%
- Через 6 месяцев: ≥ 75%
- Через 12 месяцев: ≥ 90%

**Источник:** журнал решений ОД (ведёт сама ОД).

**Периодичность:** ежемесячно.

### 4.3 Скорость принятия операционных решений

**Что:** среднее время от появления вопроса до принятия решения.

**Норма:** ≤ 48 часов для большинства оперативных вопросов.

**Источник:** трекер задач.

**Периодичность:** еженедельно.

**Что значит:** если решения «висят» неделями — ОД не работает.

### 4.4 Эскалации к собственнику

**Что:** сколько раз в неделю ОД эскалирует к собственнику.

**Норма:**
- В первые 3 месяца: 5-8 в неделю (нормально, ОД учится)
- Через 6 месяцев: 2-3 в неделю
- Через 12 месяцев: 1-2 в неделю (только серьёзные)

**Источник:** чат «ОД-собственник».

**Периодичность:** еженедельно.

---

## КЛАСТЕР 5. Стратегические инициативы

### 5.1 Количество реализованных стратегических инициатив

**Что:** сколько стратегических задач из плана собственника ОД реализовала за период.

**Норма:** ≥ 80% запланированных инициатив реализовано в срок.

**Источник:** трекер стратегических инициатив.

**Периодичность:** ежеквартально.

### 5.2 Запуск новых продуктов

**Что:** успешные запуски новых продуктов (например, летний лагерь, Иннополис, новые курсы).

**Норма:** ≥ 1 успешный запуск в год.

**Источник:** журнал продуктов.

**Периодичность:** ежегодно.

### 5.3 Открытие новых городов / точек

**Что:** при наличии планов на расширение — успешные открытия.

**Норма:** зависит от стратегии.

**Источник:** оргструктура.

**Периодичность:** по факту.

---

## КЛАСТЕР 6. Качество регламентов и стандартов

### 6.1 Актуальность регламентов

**Что:** все ключевые регламенты обновлены не позднее, чем 6 месяцев назад.

**Норма:** 100%.

**Источник:** даты последнего обновления документов в Drive.

**Периодичность:** ежемесячно.

### 6.2 Аудиты исполнения стандартов

**Что:** проведённые ОД (или подчинёнными) аудиты в филиалах: соблюдают ли стандарты.

**Норма:** ≥ 1 аудит на филиал в квартал.

**Источник:** журнал аудитов.

**Периодичность:** ежеквартально.

### 6.3 Доля филиалов, прошедших аудит «без серьёзных нарушений»

**Норма:** ≥ 80%.

**Источник:** отчёты по аудитам.

**Периодичность:** ежеквартально.

## ИНСТРУМЕНТЫ И ИСТОЧНИКИ ДАННЫХ — сводно

| Источник | Что лежит | Кто заполняет | Кто читает ОД |
|---|---|---|---|
| **CRM s20.online** | Все клиенты, ученики, статусы — по всей сети | Команда + центральные функции | Да, ежедневно |
| **Бухгалтерия / 1С** | Выручка, расходы, ФОТ, P&L всей сети | Бухгалтер | Да, на чтение |
| **Сводный дашборд сети** | Все KPI всех ролей и всех городов | Всё центральное + ОД | Да, ежедневно |
| **Дашборды подчинённых ролей** | Детальные KPI методиста, фермеров, менеджеров развития, РФ | Соответствующие роли | Да, еженедельно |
| **Журнал решений ОД** | Все принятые операционные решения | Сама ОД | Да |
| **Трекер стратегических инициатив** | Стратегические задачи + статус | Сама ОД | Да |
| **Журнал аудитов** | Аудиты в филиалах, нарушения, корректировки | ОД + подчинённые | Да |
| **HR-учёт / 1С (кадры)** | Найм, увольнения, текучка | HR | Да, ежемесячно |
| **Чат «ОД-собственник»** | Все эскалации, дайджесты | ОД + собственник | Да, ежедневно |
| **Чат «ОД-РФ»** | Групповые синки, объявления | Все РФ + ОД | Да, ежедневно |
| **Чат «ОД-центральные функции»** | Координация с методистом, РОП, РОС, маркетологом, HR | Все центральные + ОД | Да, ежедневно |

## ЕСЛИ НА КАКОЙ-ТО ИЗ ИСТОЧНИКОВ НЕТ ДОСТУПА

**Минимально критичные (без них роль не существует):**
- CRM s20.online (полный доступ)
- Бухгалтерия (полный доступ на чтение, желательно — на запись для коррекций)
- Сводный дашборд сети
- Все дашборды подчинённых ролей

**Если ОД не имеет полного доступа к финансам сети** — это саботаж роли. ОД без P&L — это исполнитель.

## КАК ОТЛИЧИТЬ «ОД РАБОТАЕТ» ОТ «ОД ЕСТЬ ФОРМАЛЬНО»

**Признаки реально работающего ОД:**
- Собственник 70%+ времени на стратегии (не на оперативке)
- Все РФ обращаются с вопросами к ОД, не к собственнику
- Все центральные функции принимают решения через ОД
- Управленческая команда видит ОД как «понятного партнёра»
- ОД проактивно ставит задачи, не «реагирует»

**Признаки «ОД формально, а реально — исполнитель» (текущая ситуация):**
- Собственник продолжает участвовать в оперативке
- Команда обращается к собственнику напрямую, ОД получает решения «вторым лицом»
- ОД не принимает финансовых решений
- ОД не имеет реального влияния на стратегию
- ОД не «защищает» свои решения перед собственником

**Если 2+ признака из второй группы — роль не работает.** Срочно — корректировка через программу адаптации (см. [`02-sistema-adaptacii.md`](02-sistema-adaptacii.md)).

---

## СВЯЗЬ С ОБЩИМ ДАШБОРДОМ СЕТИ

ОД — главный пользователь общего дашборда сети. См. [`../../07-dashboard-metrik.md`](../../07-dashboard-metrik.md).

ОД должна знать **каждый показатель в дашборде наизусть** и понимать, какой тренд за последние 3 месяца. Это не «пересмотр в Excel перед встречей» — это **постоянная оперативная картина**.

## ОСОБЕННОСТЬ — РАЗДЕЛЕНИЕ ПОКАЗАТЕЛЕЙ ОД И СОБСТВЕННИКА

| Показатель | За кого |
|---|---|
| Прибыль всей сети | ОД (исполнение) + собственник (стратегия) |
| Маржа сети | ОД (главный показатель эффективности) |
| Базовый рост (10-15% в год) | ОД |
| Стратегический рост (открытие нового города) | Собственник |
| Удержание ключевых руководителей | ОД |
| Найм нового топ-менеджера | ОД (поиск) + собственник (финал) |
| Стратегия на 1-3 года | Собственник |
| Реализация стратегии | ОД |
| Открытие нового бизнеса (вне KIBERone) | Собственник |
| Качество операционки KIBERone | ОД |

Это разделение нужно зафиксировать в начале сотрудничества с ОД и **не размывать** — иначе либо ОД делает чужую работу, либо собственник продолжает делать «свою и чужую».
