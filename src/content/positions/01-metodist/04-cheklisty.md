# Чек-листы методиста

Все чек-листы — рабочий инструмент. Печатаются на бумагу или открываются в Google Sheets/Notion. **Заполнение фиксируется** (галочка с датой и подписью), хранится минимум 6 месяцев.

Чек-листы — это не «формальность для отчёта». Это страховка: если что-то идёт не так, сразу видно, какой пункт не закрыли, и где сломалось.

---

## ЧЕК-ЛИСТ 1. Ежедневный (заполняет сам методист в конце рабочего дня)

**Дата:** _______ **Методист:** _______

### Уроки
- [ ] Посетил минимум 2 урока (физически или по видео)
- [ ] Заполнил чек-лист качества по каждому уроку (см. чек-лист 4)
- [ ] Отправил короткий фидбэк тьютору в течение часа после урока

### Коммуникация
- [ ] Прочёл чат с тьюторами (с утра)
- [ ] Ответил на все вопросы тьюторов в течение 4 часов
- [ ] Прочёл чат фермеров (на предмет жалоб родителей по урокам)

### Аналитика
- [ ] Внёс данные за день в дашборд (см. `05-pokazateli-i-istochniki.md`)
- [ ] Записал заметки в дневник методиста (свободная форма)

### Тревожные сигналы (см. `06-ruka-na-pulse.md`)
- [ ] Проверил тревожные сигналы за вчера
- [ ] Эскалировал к собственнику, если что-то требует решения

### Следующий день
- [ ] Запланировал день: какие уроки буду смотреть, какие встречи, на что фокус

**Если хотя бы одна галочка не стоит** — день не закрыт. На следующий день — приоритет догнать.

## ЧЕК-ЛИСТ 2. Недельный (заполняет в пятницу)

**Неделя с _____ по _____** **Методист:** _______

### Объём работы
- [ ] Просмотрено минимум 10 уроков за неделю
- [ ] Проведена методическая встреча сети (понедельник)
- [ ] Сделано минимум 3 индивидуальных 1-1 с тьюторами в зоне риска
- [ ] Заполнен дашборд за неделю
- [ ] Отчёт за неделю отправлен собственнику

### Качество работы
- [ ] Все тьюторы получили фидбэк по своим урокам (никто не пропущен)
- [ ] Если есть тьютор с 3+ низкими баллами подряд — назначен план развития
- [ ] Если есть жалоба от родителя — расследована и закрыта (или есть план)
- [ ] Все эскалации к собственнику закрыты или есть статус

### Команда
- [ ] Знаю, кто из тьюторов «в зелёной зоне», кто — «в жёлтой/красной»
- [ ] Знаю текущий статус каждого новичка на онбординге
- [ ] Связь с руководителями филиалов — синки прошли по плану

### Документы
- [ ] Заметки за неделю систематизированы (не валяются)
- [ ] Если разработал что-то новое (шаблон, материал) — лежит в общем хранилище

### Следующая неделя
- [ ] План на следующую неделю составлен и согласован с собственником
- [ ] Командировка (если есть) — забронирована, согласована с филиалом

## ЧЕК-ЛИСТ 3. Ежемесячный (заполняет в последнюю пятницу месяца)

**Месяц _____ ** **Методист:** _______

### Метрики (см. `07-kpi-i-mesyachnye-metriki.md`)
- [ ] Сводный балл качества по сети за месяц — посчитан, в дашборде
- [ ] Текучка тьюторов за месяц — посчитана
- [ ] Доля тьюторов с баллом < 7 — посчитана
- [ ] Прогресс детей по программе — собран по филиалам
- [ ] Опрос родителей (если в этом месяце по графику) — проведён, результаты обработаны
- [ ] Опрос тьюторов (мини, 3 вопроса) — проведён

### Аттестации и онбординг
- [ ] Все запланированные аттестации новых тьюторов проведены
- [ ] Решения по новичкам приняты и оформлены
- [ ] План онбординга на следующий месяц (если есть новички) готов

### Командировки
- [ ] Минимум 1 командировка по графику проведена
- [ ] Отчёт по командировке оформлен и передан собственнику
- [ ] Договорённости с филиалом — поставлены на контроль

### Презентация собственнику
- [ ] Презентация результатов месяца подготовлена (1-2 страницы)
- [ ] Встреча с собственником (1 час) — проведена
- [ ] План на следующий месяц — согласован

### Самооценка
- [ ] Заполнил мини-опрос для себя: что получилось, что — нет, что скорректировать в следующем месяце

## ЧЕК-ЛИСТ 4. Качество урока (заполняется на каждом просмотре)

**Дата:** _____ **Тьютор:** _____ **Группа:** _____ **Тема урока:** _____ **Методист:** _____

### Структура урока (3 пункта)
1. [ ] Урок начался вовремя (±5 минут)
2. [ ] Тьютор обозначил цель урока в первые 5 минут
3. [ ] Урок завершился рефлексией / показом результатов

### Вовлечённость детей (3 пункта)
4. [ ] Все дети были вовлечены в первые 10 минут (нет «отключённых»)
5. [ ] Тьютор реагировал на индивидуальные затруднения детей (помогал, не вёл «за руку»)
6. [ ] Тьютор удерживал дисциплину без крика и репрессий

### Содержание (3 пункта)
7. [ ] Содержание соответствует утверждённой программе (без отсебятины)
8. [ ] Объяснение понятное для возраста группы (нет «слишком сложно/слишком просто»)
9. [ ] Использовал инструменты урока (Scratch / Roblox / другое) уверенно, без сбоев

### Обратная связь (1 пункт)
10. [ ] Тьютор дал каждому ребёнку конкретную обратную связь (не общее «молодец»)

**Итого баллов: ___ / 10**

**Шкала:**
- 9-10 — отличный урок, тьютор в зелёной зоне
- 7-8 — нормальный урок, есть зоны роста
- 5-6 — есть проблемы, нужен план развития (1-1, контрольный просмотр через неделю)
- ≤ 4 — критический урок, эскалация к собственнику + план «исправить или замена»

**Заметки методиста (свободная форма, что улучшить):**
_______________________________________________

**Фидбэк тьютору отправлен:** [ ] Да, в _______ (Telegram/чат) [ ] Нет, причина: _______

## ЧЕК-ЛИСТ 5. Аттестация тьютора (раз в 6 месяцев)

**Тьютор:** _____ **Дата:** _____ **Стаж:** _____ месяцев

### Метрики за последние 6 месяцев
- [ ] Средний балл качества урока: _____ (норма ≥ 7.5)
- [ ] Посещаемость в его группах: _____ % (норма ≥ 80%)
- [ ] Жалоб от родителей: _____ (норма ≤ 2 за 6 месяцев, если они разрулены)
- [ ] Опрос родителей по тьютору: _____ /5 (норма ≥ 4.3)
- [ ] Доля учеников, продливших обучение: _____ % (норма ≥ 70%)

### Открытый урок (демо)
- [ ] Проведён в присутствии методиста + руководителя филиала + другого опытного тьютора
- [ ] Балл по чек-листу 4: _____ /10 (норма ≥ 8)

### Самооценка тьютора
- [ ] Тьютор заполнил мини-опрос: что получается, что хочет улучшить, какие группы любит, какие — нет

### Решение по итогам
- [ ] **Зелёная зона:** продолжаем, рассмотреть повышение / роль наставника
- [ ] **Жёлтая зона:** план развития на 3 месяца + контрольная аттестация через 3 месяца
- [ ] **Красная зона:** план «исправить или замена» — 1 месяц на доказать, или прощаемся

**Подписи:** методист _____ , руководитель филиала _____ , собственник (для красной зоны) _____

## ЧЕК-ЛИСТ 6. Командировка в филиал (заполняется до и после)

### ДО командировки
- [ ] Согласована дата с руководителем филиала / администратором (минимум за 7 дней)
- [ ] Бронирование (билеты, гостиница) — оформлено, не позднее чем за 5 дней
- [ ] Запрошен список тьюторов и их расписание на дни командировки
- [ ] Сформулирован фокус командировки: что хочу проверить / решить (3-5 пунктов)
- [ ] Подготовлены чек-листы (распечатаны или открыты на ноутбуке)
- [ ] Согласована повестка финальной встречи с руководителем филиала

### В первый день
- [ ] Встреча с руководителем филиала / администратором — сводка ситуации
- [ ] Посещение минимум 2 уроков
- [ ] Заполнены чек-листы качества по каждому

### Во второй день
- [ ] Просмотрено минимум 4 урока
- [ ] Минимум 3 индивидуальных 1-1 с тьюторами (по 30 минут)
- [ ] Группой разбор для тьюторов филиала (1 час, в конце дня)

### В третий день
- [ ] Минимум 2 контрольных урока (для сравнения с днём 2)
- [ ] Финальная встреча с руководителем филиала — договорённости с дедлайнами
- [ ] Список договорённостей — записан, отправлен письменно (Telegram/email)

### ПОСЛЕ командировки (в течение 3 дней)
- [ ] Отчёт по командировке оформлен (3-5 страниц): что увидел, что договорились, кто ответственный, к какому сроку
- [ ] Отчёт передан собственнику
- [ ] Список договорённостей внесён в общий контроль (дашборд / трекер задач)
- [ ] Тьюторы филиала получили письменное summary разбора (не пересказ устный, а текст)

## ЧЕК-ЛИСТ 7. Онбординг нового тьютора (методист как куратор)

См. полную программу: [`../../08-onboarding-tutora.md`](../../08-onboarding-tutora.md). Здесь — короткий чек-лист методиста по контролю.

### Неделя 1
- [ ] Подготовлен Welcome Pack для нового тьютора (см. список в файле онбординга)
- [ ] Назначен наставник из числа опытных тьюторов в этом филиале
- [ ] Наставничество включено в задачи наставника на период онбординга
- [ ] Методист провёл вводную встречу с новичком (день 1, 2 часа)
- [ ] График наблюдения за уроками наставника составлен (минимум 8 уроков)

### Неделя 2
- [ ] Методист участвует в аттестации недели 2 (демо-урок новичка)
- [ ] Заполнен чек-лист 4 (качество урока)
- [ ] Решение «годен / нужна ещё неделя» — оформлено

### Неделя 3
- [ ] Методист встретился с новичком 1-1 (середина недели, 30 мин)
- [ ] Контрольный просмотр одного урока новичка (тьютор уже ведёт сам)
- [ ] Сводка от наставника получена

### Неделя 4
- [ ] Финальная аттестация (открытый урок + чек-лист)
- [ ] Опрос родителей (анонимный) — проведён
- [ ] Решение по новичку оформлено и передано собственнику
- [ ] Если «годен» — план сопровождения на месяцы 2-3 готов

## КАК ХРАНЯТСЯ ЧЕК-ЛИСТЫ

**Где:**
- Google Drive: `KIBERone Management / Чек-листы методиста / [год] / [месяц] /`
- Каждый чек-лист — отдельный файл с датой в названии
- Раз в квартал — архивируется в общий ZIP

**Доступ:**
- Методист — читает и пишет
- Собственник / операционный директор — читает все
- Руководители филиалов — читают чек-листы по своему филиалу

**Срок хранения:**
- Чек-листы качества урока — 12 месяцев (для тренда)
- Аттестации — бессрочно (для KPI и кадровых решений)
- Отчёты по командировкам — 24 месяца

**Правило:** если чек-лист не заполнен или потерян, считается, что соответствующая работа НЕ была сделана. Это защита от «я делал, но не записывал».
